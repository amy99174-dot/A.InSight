# 問題檢查報告 (Problem Diagnosis Report)

### 🔴 核心問題：為什麼「介紹頁面」沒有出現？
經過比對 **Refactor 分支** 與 **目前 UI 分支** 的程式碼，我們發現「介紹頁面消失」其實是由兩個原因共同造成的，這並非單純的程式碼錯誤，而是邏輯與 UI 整合時的落差。

#### 1. 邏輯觸發了「跳過機制」 (Logic Issue)
在 `refactor-api` 的原始邏輯中，有一段「錯誤處理機制」。
當系統無法連線到 AI (例如金鑰遺失或網路錯誤) 時，它會進入 **「備用模擬模式 (Fallback Mode)」**。
- **Refactor 的設定是**：如果進入模擬模式，為了避免沒有語音檔卻要在畫面乾等，程式碼會直接 `setStep(STEPS.FOCUSING)`，也就是 **強制跳過** `LISTEN` (聆聽/介紹) 階段。
- **結果**：因為剛才您的 API 連線失敗 (金鑰問題)，系統判定進入模擬模式，所以依照邏輯 **直接跳過了介紹頁面**。

#### 2. UI 顯示缺漏 (Rendering Issue)
即便流程成功走完最後的 `REVEAL` (揭示) 階段，目前的 `ScannerUI.tsx` 元件也存在一個顯示漏洞。
- **Refactor 分支**：主程式直接控制畫面，會算出一個 `KeyholeViewer` 來顯示最終的歷史修復圖片。
- **UI 分支**：我们将畫面整併進 `ScannerUI`，但是 `ScannerUI` 的 `REVEAL` 區塊 **忘了把 `resultImage` (結果圖) 畫出來**。它只顯示了文字框和邊框，卻沒有顯示那張圖片。

### 🔍 差異比對摘要 (Diff Summary)

| 檢查項目 | Refactor 分支 (原始邏輯) | UI 分支 (目前狀態) | 影響結果 |
| :--- | :--- | :--- | :--- |
| **API 失敗處理** | 失敗時跳過 `LISTEN` 階段 | **繼承了此邏輯** (完全相同) | 因 API 連線失敗，導致介紹頁面被跳過。 |
| **結果圖顯示** | 在 `page.tsx` 直接 render `<img />` | 傳送給 `ScannerUI` 但 **未被繪製** | 就算流程跑完，最後也是黑畫面，看不到圖。 |
| **程式碼矛盾** | 無 | 無 | 邏輯本身沒有矛盾，是「錯誤後的行為」符合預期但使用者未察覺。 |

### ✅ 結論與建議
這個問題並非「改壞了」，而是：
1.  **觸發了隱藏邏輯**：因為沒設 API Key -> 觸發錯誤 -> 邏輯設計為「跳過介紹」。
2.  **UI 缺少實作**：新的 UI 元件漏寫了圖片顯示的代碼。

**不需擔心程式碼混亂**，目前的結構是清晰的。只要修正 API Key 讓它走「成功路徑」，介紹頁面就會出現；同時只需在 UI補上圖片顯示即可。
